1. Mis on versioonihaldus? 
Versioonihaldus on süsteem, mis jälgib ja haldab koodi või failide muudatusi aja jooksul. See võimaldab:

Salvestada muudatuste ajalugu.
Töödelda sama projekti erinevaid versioone paralleelselt.
Tagasi kerida vanema versiooni juurde, kui midagi läheb valesti. Näiteks populaarsed versioonihaldustööriistad on Git, Subversion (SVN) ja Mercurial.
2. Kuidas hallata koodi? 
Koodi haldamiseks kasutatakse versioonihaldustööriistu, näiteks Git. Põhiprintsiibid:

Loo repositoorium:
bash
Копировать код
git init
Lisa failid jälgimise alla:
bash
Копировать код
git add [failinimi]
Loo commit (salvesta muudatused):
bash
Копировать код
git commit -m "Muudatuse kirjeldus"
Push (laadi kood kaug-reposse):
bash
Копировать код
git push origin [haru nimi]
Pull (tõmba värskeim kood):
bash
Копировать код
git pull

3. Kuidas luua harusid? 
Harud võimaldavad arendada projekti erinevaid osi paralleelselt, ilma et peaharu (main) kohe muutuks. Haru loomine ja haldamine Gitis:

Loo uus haru:
bash
Копировать код
git branch [haru nimi]
Näiteks:
bash
Копировать код
git branch uus-funktsioon
Mine harule üle:
bash
Копировать код
git checkout [haru nimi]
või kombineeritud:
bash
Копировать код
git checkout -b [haru nimi]
Sulata haru peaharusse (merge): Mine tagasi peaharule:
bash
Копировать код
git checkout main
Ja ühenda haru:
bash
Копировать код
git merge [haru nimi]
Kustuta haru (vajadusel):
bash
Копировать код
git branch -d [haru nimi]
